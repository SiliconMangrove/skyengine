# 工厂资产模块改造方案

## 一、改造目标

将 ConfigPanel 中的「工厂节点资产」静态统计展示区域，改造为**可交互的工厂编辑器资产面板**，支持：
1. 从资产池添加数字资产到工厂场景
2. 在 3D 场景中拖放布局（网格对齐 + 高亮）
3. 编辑完成后导出配置文件，供后续调度仿真使用
4. 编辑模式下禁止启动仿真

---

## 二、组件拆分方案

### 2.1 新组件结构

```
ConfigPanel.vue（精简，保留配置管理 + 文件上传）
└── FactoryAssetPanel.vue（新建，承载所有工厂资产业务）
    ├── 资产统计栏（改造自原紫色渐变区域）
    ├── 资产池区域（新增：下拉选择 + 预览 + 添加按钮）
    ├── 已放置资产列表（改造自原 services-list）
    └── 编辑模式工具栏（新增：进入/退出编辑、导出配置）
```

### 2.2 拆分边界

| 原位置 | 迁移到 | 说明 |
|--------|--------|------|
| ConfigPanel 内 `<!-- 工厂节点资产 -->` 整个 `section-block` | `FactoryAssetPanel.vue` | 模板 + 逻辑 + 样式全部搬出 |
| `assetsList` / `assetsStats` computed | `FactoryAssetPanel.vue` 内部计算 | 从 store 读取，不再依赖 ConfigPanel |
| `onDragStart` 方法 | `FactoryAssetPanel.vue` | 拖拽逻辑跟随资产组件走 |
| ConfigPanel 的 `defineExpose` | 仅保留 `getSelectedFile` / `resetFile` | 资产相关暴露删除 |

## 三、功能模块设计

### 3.1 模块一：资产池（Asset Pool）

**位置**：原紫色统计方框区域改造

**功能**：
- **下拉选择器**（`el-select`）：列出所有支持的数字资产类型
  - 区域类：禁区（restricted）、工位（workbench）、障碍物（obstacle）、工作区（workarea）
  - 机器类：加工机器（machine）
  - 路由点类：停靠点（dock）、路由点（route）
- **预览槽**：选中资产后，显示该资产的 3D 形状缩略预览（使用小 Canvas + Three.js 渲染缩略图，或用 CSS/SVG 图示替代）
- **添加按钮**：将选中资产添加到工厂场景中（默认位置取网格中心或第一个空位）

**数据来源**：在 store 中定义资产模板注册表：

```js
// factory.js store 新增
const ASSET_TEMPLATES = {
  zone_restricted:  { type: 'zone',    icon: '🔒', name: '禁区',   template: { type: 'restricted', area: { x:0, y:0, w:2, h:2 }, color: 'rgba(255,235,59,0.15)' } },
  zone_workbench:   { type: 'zone',    icon: '🔧', name: '工位',   template: { type: 'workbench',  area: { x:0, y:0, w:1, h:1 }, color: 'rgba(0,136,204,0.15)' } },
  zone_obstacle:    { type: 'zone',    icon: '🚫', name: '障碍物', template: { type: 'obstacle',   area: { x:0, y:0, w:1, h:1 }, color: 'rgba(102,51,51,0.15)' } },
  zone_workarea:    { type: 'zone',    icon: '📦', name: '工作区', template: { type: 'workarea',   area: { x:0, y:0, w:3, h:2 }, color: 'rgba(0,170,102,0.15)' } },
  machine:          { type: 'machine', icon: '⚙️', name: '机器',   template: { size: [2,2], status: 'IDLE' } },
  waypoint_dock:    { type: 'waypoint',icon: '🚪', name: '停靠点', template: { type: 'dock' } },
  waypoint_route:   { type: 'waypoint',icon: '📍', name: '路由点', template: { type: 'route' } },
}
```

### 3.2 模块二：已放置资产列表

**位置**：原 `services-list-container` 区域改造

**功能**：
- 展示当前配置中所有已放置的数字资产（名称 + 类型图标 + 统计）
- **编辑模式下**：
  - 每个资产行显示**重命名输入框**（`el-input`，默认 readonly，点击可编辑）
  - 右侧增加**删除按钮**（从场景和配置中移除该资产）
  - 添加新资产后实时追加到列表底部
- **非编辑模式**：
  - 保持原有拖拽节点行为（`draggable="true"`）
  - 仅展示，不可增删改

### 3.3 模块三：编辑模式控制

**新增状态**：`isEditMode`（`ref(false)`）

**编辑模式切换按钮**：
- 位于资产面板标题行右侧
- 点击「进入编辑」→ `isEditMode = true`，触发事件 `@edit-mode-change(true)`
- 点击「退出编辑」→ `isEditMode = false`，触发事件 `@edit-mode-change(false)`

**编辑模式约束**：
- 进入编辑时 emit `edit-mode-change(true)`，父组件（FactoryPlayerSSE）接收后禁用 ControlPanel 的播放按钮
- 退出编辑时恢复

**导出配置按钮**：
- 仅在编辑模式下显示
- 点击后将当前 store 中的配置序列化为 JSON 并触发下载
- 调用现有的 `downloadTemplate` 类似逻辑，但导出当前实时配置

### 3.4 模块四：3D 场景拖放布局（FactoryVisualization3D 改造）

**核心交互**：
1. 资产面板中选中一个已添加但未定位的资产 → 鼠标变为"放置"光标
2. 在 3D 场景中点击网格 → 资产放置到点击位置（自动对准最近网格点）
3. 已放置的资产可拖动 → 按住拖动重新定位（实时更新网格对齐位置）
4. 放置/拖动时高亮显示目标网格位置（半透明预览方块）

**技术实现**：
- 使用 Three.js `Raycaster` 检测鼠标与地面平面的交点
- 将交点坐标转换为网格坐标 `snapToGrid(worldPos)`
- 编辑模式下启用 `TransformControls` 或自定义拖拽逻辑
- 高亮使用临时 Mesh（`MeshBasicMaterial`，transparent + opacity）

**FactoryVisualization3D 新增 props**：
```js
props: {
  // ...existing
  editMode: { type: Boolean, default: false },
}
```

**新增 emit 事件**：
```js
emit('asset-placed', { assetId, assetType, gridX, gridY })
emit('asset-moved', { assetId, gridX, gridY })
```

## 四、数据流设计

```
FactoryAssetPanel                     FactoryVisualization3D
       │                                        │
       │  store.addAsset(templateId)            │
       ├──────────► factory.js store ──────────►│  buildStaticElements() 重建
       │                                        │  新增资产渲染到场景
       │                                        │
       │  editMode prop ──────────────────────► │  启用 Raycaster 拖放
       │                                        │
       │  ◄──── emit('asset-placed', {...}) ────│  用户在场景中点击放置
       │                                        │
       │  store.updateAssetPosition(...)         │
       ├──────────► factory.js store ──────────►│  位置更新，重新渲染
       │                                        │
       │  store.exportConfig() → JSON 下载       │
```

## 五、Store 扩展（factory.js）

需要新增以下方法：

```js
// 添加资产（从模板创建）
function addAssetFromTemplate(templateId: string): string
// 返回新资产的 ID

// 更新资产位置
function updateAssetPosition(assetType: 'zone'|'machine'|'waypoint', assetId: string, gridX: number, gridY: number): void

// 重命名资产
function renameAsset(assetType: 'zone'|'machine'|'waypoint', assetId: string, newName: string): void

// 删除资产
function removeAsset(assetType: 'zone'|'machine'|'waypoint', assetId: string): void

// 导出当前配置为 JSON
function exportCurrentConfig(): object
```

## 六、样式方案

- `FactoryAssetPanel.vue` 使用独立 CSS 文件：`components/styles/FactoryAssetPanel.css`
- 复用现有 `theme.css` 中的 CSS 变量（暗色科技风格）
- 资产池区域保持紫色渐变背景（`linear-gradient(135deg, #667eea, #764ba2)`），内部改为下拉框 + 预览布局
- 编辑模式下资产列表项增加左侧可编辑指示器（蓝色边框 + 编辑图标）
- 网格高亮使用 `rgba(102, 126, 234, 0.3)` 半透明色块

## 七、实施步骤

### Phase 1：组件拆分（基础）
1. 创建 `FactoryAssetPanel.vue`，将 ConfigPanel 中工厂资产 section-block 搬出
2. ConfigPanel 改为引入 `FactoryAssetPanel` 子组件
3. RightSidePanel 透传不需要改动（ConfigPanel 内部组件变更对外透明）
4. 验证拆分后原有功能不变（统计展示、拖拽节点、配置切换）

### Phase 2：Store 扩展
5. 在 `factory.js` 中新增 `ASSET_TEMPLATES` 常量
6. 实现 `addAssetFromTemplate` / `updateAssetPosition` / `renameAsset` / `removeAsset` / `exportCurrentConfig`
7. 确保 `_persistConfigs()` 在每次变更后调用

### Phase 3：资产池 UI
8. 改造紫色统计区域为资产池（`el-select` + 预览 + 添加按钮）
9. 统计信息移至资产池下方，紧凑展示
10. 实现添加逻辑：选择模板 → 调用 store → 资产列表实时更新

### Phase 4：编辑模式
11. 新增 `isEditMode` 状态 + 标题栏编辑按钮
12. 编辑模式下资产列表支持重命名 + 删除
13. emit `edit-mode-change` 事件，父组件接收后禁用仿真控制
14. 实现导出配置按钮（JSON 下载）

### Phase 5：3D 场景拖放
15. FactoryVisualization3D 新增 `editMode` prop
16. 实现 Raycaster 地面点击检测 + 网格对齐
17. 实现放置高亮预览（hover 时显示半透明方块）
18. 实现已放置资产拖动重定位
19. emit `asset-placed` / `asset-moved` 事件，联动 store 更新
