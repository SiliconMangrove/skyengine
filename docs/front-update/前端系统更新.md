思考了下，当前前端系统还是存在一些问题，
举例来说，
1. 不够着眼于工厂系统本身，周围的各种panel占用位置太多了，需要更新整合。

方案：创建一个可拖动的panel，将现在所有的panel（包括：控制面板，仿真工作台，数据与配置，工厂资产，关键指标，日志）等这几个panel，将工厂本身尽可能地展示出来。

2. 后续需要添加新的panel，将调度事件等展示出来。并且进行一些细节的优化，举例来说：

AGV 加路径线（蓝/绿/黄三色区分执行中/已完成/待执行），设备和 AGV 头顶加状态浮标，这两件事视觉冲击最大、代码量最少。工件/托盘在设备间流动的动画可以做简化版，不需要真实物理，用一个小方块沿路径插值移动就够。

调度日志流是性价比最高的——一个滚动列表，把后端每次分配决策打印出来，几乎零工作量但答辩时演示效果很好。热力图叠加层也不难，用 Canvas 在地面上绘制颜色渐变就行。调度决策面板（候选 AGV + 评分 + 最终选择）这个要后端配合吐数据，值得做。
