## 一、系统定位

### 1.1 双项目关系

天工系统由两个独立仓库协作构成，遵循**两层解耦**架构原则：

| 项目 | 路径 | 定位 | 职责边界 |
|------|------|------|---------|
| **天工平台** | `grouppro/skyengine` | 用户交互层（Platform） | 前端可视化 + 后端网关 + 工厂生命周期管理，**不含任何调度算法** |
| **天工引擎** | `finalpro/SkyEngine` | 算法执行层（Engine） | 仿真环境 + 调度算法 + RL 训练 + 求解器微服务，**不含任何业务界面** |

```
┌─────────────────────────────────────────────────────┐
│                    用户浏览器                         │
│         Vue 3 + Three.js 3D + Element Plus           │
└──────────────────────┬──────────────────────────────┘
                       │ SSE / REST
┌──────────────────────▼──────────────────────────────┐
│              天工平台（grouppro/skyengine）             │
│  FastAPI 后端 + FactoryProxy 抽象 + Docker 网关       │
│  ├── StaticFactoryProxy   → 内置演示数据              │
│  ├── PacketFactoryProxy   → 翼辉电池产线（内嵌仿真）    │
│  └── DockerProxy          → 按需启动引擎容器           │
└──────────────────────┬──────────────────────────────┘
                       │ Docker SDK / HTTP
┌──────────────────────▼──────────────────────────────┐
│            天工引擎（finalpro/SkyEngine）              │
│  GridFactoryEnv + Coordinator (FJSP + MAPF + Assign) │
│  ├── engine 容器  → sim_server.py (FastAPI + SSE)    │
│  ├── fjsp 容器    → PSO / DE / DRL / Best 微服务     │
│  └── mapf 容器    → A* / MAPF-GPT 微服务             │
└─────────────────────────────────────────────────────┘
```

### 1.2 各自演进方向

**天工平台**的演进方向：**"可扩展的制造仿真可视化平台"**
- 从"演示 Demo"向"可交付产品"演进
- 核心价值：统一的 3D 可视化工作台 + 多工厂类型即插即用 + 算法实验管理
- 交付形态：docker-compose 一键部署的 Web 应用

**天工引擎**的演进方向：**"研究级 FJSP+MAPF 联合求解引擎"**
- 从"实验脚本"向"标准化仿真服务"演进
- 核心价值：耦合调度求解 + RL 训练管线 + 求解器微服务化
- 交付形态：docker-compose 编排的微服务集群（engine + fjsp + mapf）
