# 演进方案 — DockerProxy + sim_server 对齐

## 核心思路

**当前问题**: DockerProxy 收到前端完整 JSON 配置后，自行拆解成 `fjsp_instance`/`mapf_instance`/`mapf_service_url` 等字段传给 sim_server。sim_server 又要从文件系统重新读取解析。两处解析逻辑不一致，容易出错。

**改进方案**: DockerProxy 把前端传来的配置 JSON **原样转发**给 sim_server，sim_server 用和 `use_io.py` 同款的解析逻辑直接处理。两端共用一套解析逻辑。

---

## 1. 数据流（改进后）

```
前端 GridFactoryManage.vue
  │ POST /factory/config/upload
  │ body: { filename: "xx", config: { topology, agvs, jobs, ... } }
  ▼
server.py
  │ proxy.set_config(config)          ← config 就是完整 JSON
  ▼
DockerProxy.set_config(config)
  │ self._config = config             ← 原样存储
  ▼
DockerProxy.start()
  │ POST engine /sim/create
  │ body: {
  │     "config": self._config,       ← 原样转发
  │     "fjsp_algorithm": "pso",
  │     "mapf_algorithm": "astar",
  │     "solver_assign": "nearest"
  │ }
  ▼
sim_server POST /sim/create
  │ create_env_from_config(config)    ← 用 use_io.py 同款逻辑解析
  │ Coordinator(job_solver="http", route_solver="http", ...)
  ▼
仿真运行
```

## 2. DockerProxy 改动

### 2.1 `start()` 中的 sim_config 构造

**当前** (`DockerProxy.py:280-293`):
```python
sim_config = {
    "fjsp_instance": self._config.get("fjsp_instance", "J10P5M6.json"),
    "mapf_instance": self._config.get("mapf_instance", ""),
    "solver_job": "http",
    "solver_route": "http",
    "solver_assign": self._algorithm_parts.get("assigner", "random"),
    "mapf_service_url": "http://mapf:8001",
    "fjsp_service_url": "http://fjsp:8002",
    "num_agv": self._config.get("num_agv", 4),
    "seed": self._config.get("seed", 42),
    "max_steps": self._config.get("max_steps", 1000),
    ...
}
```

**改为**:
```python
sim_config = {
    "config": self._config,  # 前端完整 JSON，原样转发
    "fjsp_algorithm": self._algorithm_parts.get("fjsp", "pso"),
    "mapf_algorithm": self._algorithm_parts.get("mapf", "astar"),
    "solver_assign": self._algorithm_parts.get("assigner", "nearest"),
}
```

不再需要:
- `fjsp_instance` / `mapf_instance` (文件名)
- `mapf_service_url` / `fjsp_service_url` (sim_server 内部约定)
- `solver_job` / `solver_route` (固定为 HTTP)
- `num_agv` / `seed` / `max_steps` 等 (都在 config 里了)

### 2.2 其他方法不变

`initialize()`, `pause()`, `reset()`, `stop()`, `cleanup()` 以及所有 SSE 透传逻辑**不变**。

## 3. sim_server 改动

### 3.1 删除旧的文件解析函数

删除 `parse_fjsp_instance()`, `parse_mapf_instance()`, 旧的 `create_env()`。

### 3.2 新增 `create_env_from_config()` — 和 use_io.py 同款逻辑

用 `sky_executor` 的 import 替代 `joint_sim`：

```python
from sky_executor.grid_factory.factory.Utils.structure import MachineConfig, JobConfig
from sky_executor.grid_factory.factory.grid_factory_env import GridFactoryEnv
from pogema import GridConfig


def create_env_from_config(config: dict) -> GridFactoryEnv:
    """从 JSON 配置创建 GridFactoryEnv — 和 use_io.py 同款解析逻辑"""

    # ---- GridConfig ----
    topology = config.get("topology", {})
    agvs = config.get("agvs", [])
    grid_width = topology.get("gridWidth", 20)
    grid_height = topology.get("gridHeight", 20)
    grid_size = max(grid_width, grid_height)
    num_agents = len(agvs)
    agents_start_xy = [tuple(agv["initialLocation"]) for agv in agvs]

    grid_cfg = GridConfig(
        size=grid_size,
        num_agents=num_agents,
        density=0.0,
        seed=42,
        max_episode_steps=256,
        obs_radius=5,
        on_target="restart",
        agents_xy=agents_start_xy if agents_start_xy else None,
        targets_xy=agents_start_xy if agents_start_xy else None,
    )

    # ---- MachineConfig ----
    machines = topology.get("machines", {})
    machine_positions = [tuple(m["location"]) for m in machines.values()]

    machine_cfg = MachineConfig(
        num_machines=len(machines),
        strategy="custom",
        custom_positions=machine_positions,
        seed=42,
    )

    # ---- JobConfig ----
    jobs = config.get("jobs", {})
    job_list = jobs.get("job_list", [])

    custom_jobs = []
    for job in job_list:
        job_ops = []
        for op in job.get("operations", []):
            machine_id = op.get("machine_id", 0)
            duration = op.get("duration", 1)
            machine_options = [machine_id] if isinstance(machine_id, int) else machine_id
            job_ops.append((machine_options, duration))
        custom_jobs.append(job_ops)

    job_cfg = JobConfig(
        num_jobs=len(job_list),
        min_ops_per_job=min((len(j) for j in custom_jobs), default=1),
        max_ops_per_job=max((len(j) for j in custom_jobs), default=1),
        min_proc_time=1,
        max_proc_time=1,
        machine_choices=1,
        total_machines=len(machines),
        seed=42,
        strategy="custom",
        custom_jobs=custom_jobs,
    )

    # ---- 设置机器位置为 possible_targets ----
    grid_cfg.possible_targets_xy = machine_positions

    return GridFactoryEnv(
        grid_config=grid_cfg,
        machine_config=machine_cfg,
        job_config=job_cfg,
        random_target=False,
    )
```

### 3.3 `SimulationManager.create()` 改造

```python
def create(self, config: dict):
    factory_config = config.get("config", {})
    fjsp_algo = config.get("fjsp_algorithm", "pso")
    mapf_algo = config.get("mapf_algorithm", "astar")
    assigner = config.get("solver_assign", "nearest")

    # 用同款逻辑解析配置 → 创建环境
    self.env = create_env_from_config(factory_config)
    self.obs, info = self.env.reset()

    # Coordinator: 固定用 HTTP 求解器，连接算法容器
    # 算法容器在同一 Docker 网络，用容器名 DNS 通信
    self.coordinator = Coordinator(
        job_solver="http",
        route_solver="http",
        assigner=assigner,
        job_solver_kwargs={
            "service_url": "http://skyengine-online-fjsp:8002",
            "algorithm": fjsp_algo,
        },
        route_solver_kwargs={
            "service_url": "http://skyengine-online-mapf:8001",
            "pogema_env": self.env.pogema_env,
        },
    )
    self.max_steps = 1000
    self.step = 0
```

### 3.4 不再需要的环境变量

删除 `create()` 中设置 `MAPF_SERVICE_URL` / `FJSP_SERVICE_URL` 的代码。
service_url 通过 `solver_kwargs` 显式传入。

## 4. 对照表

| 项目 | 当前 | 改进后 |
|------|------|--------|
| DockerProxy → sim_server 传的数据 | 拆解后的字段（fjsp_instance, mapf_service_url 等） | 前端完整 JSON + 算法参数 |
| sim_server 解析方式 | 读文件 → parse_fjsp/parse_mapf | create_env_from_config() 同款 |
| 算法容器地址 | 环境变量 MAPF/FJSP_SERVICE_URL | solver_kwargs 显式传入，固定容器名 DNS |
| Coordinator solver 选择 | solver_job="http", solver_route="http" | 同上，固定 HTTP |
| 两端解析一致性 | 不一致 | 完全一致 |
