# 0610 — Agent RAG 交互 + 历史日志系统设计

## 1. 背景与目标

当前 `AgentPanel.vue` 的分析能力完全依赖前端硬编码的规则函数（`buildEfficiencyAnalysis` 等），无实际 AI 能力。同时系统只支持实时查看当前运行状态，仿真结束后数据即丢失，无法回溯。

本次更新目标：
1. **Agent 面板接入 vLLM/RAG**：将运行时 metrics + logs 作为上下文发送给 RAG 后端，获得智能分析
2. **扩充 monitorStore 累积能力**：让现有 store 在运行期间持续积累 metrics 时间线，作为 RAG 查询的上下文
3. **后端历史日志持久化**：将仿真运行日志保存到后端，支持历史查询与对比

## 3. 扩充 monitorStore — 累积运行时数据

### 3.1 问题分析

现有 `monitorStore` 的问题：
- `events` — FIFO 只保留 50 条，旧的直接 shift 掉了
- `chartData` — 只存当前图表数据，不是时间线累积
- `keyMetrics` — 只存当前值 `{efficiency, utilization}`

**方案**: 不新建 store，直接扩充 `monitorStore`，增加 metrics 时间线累积和 RAG 上下文构建能力。

### 3.2 monitorStore 扩充内容

在现有 `stores/monitor.js` 中新增：

```javascript
// ===== 新增: 运行时数据累积 =====
const metricsTimeline = ref([])        // [{timestamp, efficiency, utilization, ...}, ...]
const MAX_METRICS_TIMELINE = 500

const runId = ref(null)                // 当前运行 ID（后端 /history 创建时返回）
const runStartTime = ref(null)         // 运行开始时间
const runMeta = ref({})                // { factoryType, algorithm, configName }

/** 开始新运行 */
function startRun(meta = {}) {
  runId.value = meta.runId || `run_${Date.now()}`
  runStartTime.value = new Date().toISOString()
  runMeta.value = meta
  metricsTimeline.value = []
  // 注意: 不清空 eventQueue，保留现有事件
}

/** 扩充 pushMetrics — 同时累积 timeline */
function pushMetrics(data) {
  // 原有逻辑: 更新图表和卡片指标
  if (data.machine) chartData.value.machine = data.machine
  if (data.agv) chartData.value.agv = data.agv
  if (data.job) chartData.value.job = data.job
  if (data.keyMetrics) {
    keyMetrics.value = { ...keyMetrics.value, ...data.keyMetrics }
  }

  // 新增: 累积到 timeline
  metricsTimeline.value.push({
    timestamp: Date.now(),
    efficiency: data.keyMetrics?.efficiency?.value,
    utilization: data.keyMetrics?.utilization?.value,
    // 保留 chart 原始数据用于深度分析
    machine: data.machine,
    agv: data.agv,
    job: data.job,
  })
  if (metricsTimeline.value.length > MAX_METRICS_TIMELINE) {
    metricsTimeline.value = metricsTimeline.value.slice(-MAX_METRICS_TIMELINE)
  }
}

/** 构造 RAG 上下文 — 从已有数据生成 */
function buildRAGContext(currentState = null) {
  // metrics 统计
  const metricStats = {}
  const recentMetrics = metricsTimeline.value.slice(-20)
  if (recentMetrics.length > 0) {
    const effs = recentMetrics.map(m => m.efficiency).filter(v => v != null && v !== '--')
    const utils = recentMetrics.map(m => m.utilization).filter(v => v != null && v !== '--')
    if (effs.length > 0) {
      metricStats.efficiency = {
        current: effs[effs.length - 1],
        avg: +(effs.reduce((a, b) => a + Number(b), 0) / effs.length).toFixed(2),
        min: Math.min(...effs.map(Number)),
        max: Math.max(...effs.map(Number)),
      }
    }
    if (utils.length > 0) {
      metricStats.utilization = {
        current: utils[utils.length - 1],
        avg: +(utils.reduce((a, b) => a + Number(b), 0) / utils.length).toFixed(2),
        min: Math.min(...utils.map(Number)),
        max: Math.max(...utils.map(Number)),
      }
    }
  }

  // 事件统计
  const eventStats = {}
  eventQueue.value.forEach(e => {
    eventStats[e.type] = (eventStats[e.type] || 0) + 1
  })

  return {
    run: {
      id: runId.value,
      start_time: runStartTime.value,
      meta: runMeta.value,
      duration: runStartTime.value
        ? `${((Date.now() - new Date(runStartTime.value).getTime()) / 1000).toFixed(0)}s`
        : '0s',
    },
    metric_stats: metricStats,
    event_stats: eventStats,
    total_events: totalEventCount.value,
    recent_metrics: recentMetrics.slice(-10),
    recent_events: eventQueue.value.slice(-20),
    current_state: currentState,  // 从 factory store 传入
  }
}

/** 清空（重置时调用） */
function clear() {
  eventQueue.value = []
  metricsTimeline.value = []
  runId.value = null
  runStartTime.value = null
  runMeta.value = {}
}
```

### 3.3 数据流示意

```
SSE /stream/state  ──▶ factory store (实时显示 + 动画)

SSE /stream/metrics ──▶ monitorStore.pushMetrics()
                         ├─ chartData (图表更新，原有逻辑)
                         ├─ keyMetrics (卡片更新，原有逻辑)
                         └─ metricsTimeline (新增：时间线累积)

SSE control event  ──▶ monitorStore.pushEvent()
                         ├─ eventQueue (UI 事件列表，原有逻辑)

play 时 ──▶ monitorStore.startRun({ factoryType, algorithm, runId })
reset/stop 时 ──▶ 可选：调后端 /history/run/{id}/complete 保存
```

### 4.2 前端 RAG 请求封装

新增 `application/frontend/src/utils/rag.js`：

```javascript
// utils/rag.js

const RAG_URL = window.__RAG_BACKEND_URL__ || import.meta.env.VITE_RAG_URL || 'http://localhost:8000'
const RAG_API_KEY = import.meta.env.RAG_API_KEY || 'token-local-dev'

/**
 * 系统提示词 — 定义 AI 的角色和能力
 */
const SYSTEM_PROMPT = `你是天工(SkyEngine)制造系统的智能分析助手。你负责分析工厂仿真运行数据，包括AGV调度、机器负载、运输任务和系统日志。

你的能力：
1. 分析 AGV 效率、利用率和空驶情况
2. 检测生产瓶颈（机器拥堵、AGV 堆叠、物流断流）
3. 分析机器负载均衡性
4. 评估运输路径和调度合理性
5. 总结日志事件并提取关键异常
6. 基于历史趋势给出优化建议

数据说明：
- grid_state.positions_xy: AGV 的 [x, y] 坐标列表
- grid_state.is_active: AGV 活跃状态列表
- machines: 机器状态字典 {id: {status, progress, ...}}
- active_transfers: 活跃运输任务列表
- metrics: 效率和利用率指标

请用中文回答，保持简洁专业。`

/**
 * 发送 RAG 查询
 * @param {string} question - 用户问题
 * @param {object} context - monitorStore.buildRAGContext() 生成的上下文
 * @param {function} onChunk - 流式回调 (chunk: string) => void
 * @returns {Promise<string>} 完整回复
 */
export async function queryRAG(question, context, onChunk = null) {
  const userMessage = JSON.stringify({
    question,
    context,
  })

  const response = await fetch(`${RAG_URL}/v1/chat/completions`, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${RAG_API_KEY}`,
    },
    body: JSON.stringify({
      model: 'qwen3.6-35b',
      messages: [
        { role: 'system', content: SYSTEM_PROMPT },
        { role: 'user', content: userMessage },
      ],
      max_tokens: 1024,
      temperature: 0.7,
      stream: !!onChunk,
    }),
  })

  if (!response.ok) {
    throw new Error(`RAG 请求失败: ${response.status}`)
  }

  // 流式响应处理
  if (onChunk && response.body) {
    const reader = response.body.getReader()
    const decoder = new TextDecoder()
    let fullContent = ''

    while (true) {
      const { done, value } = await reader.read()
      if (done) break

      const chunk = decoder.decode(value, { stream: true })
      // 解析 SSE 格式: data: {"choices":[{"delta":{"content":"..."}}]}
      for (const line of chunk.split('\n')) {
        if (!line.startsWith('data: ') || line === 'data: [DONE]') continue
        try {
          const parsed = JSON.parse(line.slice(6))
          const content = parsed.choices?.[0]?.delta?.content || ''
          if (content) {
            fullContent += content
            onChunk(content)
          }
        } catch { /* 忽略解析错误 */ }
      }
    }
    return fullContent
  }

  // 非流式响应
  const data = await response.json()
  return data.choices?.[0]?.message?.content || '无回复'
}
```

### 4.3 AgentPanel.vue 改造

改造 `AgentPanel.vue` 中的 `runAnalysis` 函数，RAG 优先 + 本地 fallback：

```javascript
// AgentPanel.vue <script setup> 改动部分

import { useMonitorStore } from '@/stores/monitor'
import { useFactoryStore } from '@/stores/factory'
import { queryRAG } from '@/utils/rag'

const monitorStore = useMonitorStore()
const factoryStore = useFactoryStore()
const isAnalyzing = ref(false)

async function runAnalysis(type, prompt) {
  const ragUrl = window.__RAG_BACKEND_URL__ || import.meta.env.VITE_RAG_URL

  if (!ragUrl) {
    // 无 RAG: 使用原有硬编码分析
    analysisResult.value = runLocalAnalysis(type, prompt)
    return
  }

  isAnalyzing.value = true
  analysisResult.value = '正在分析中...'

  try {
    const currentState = factoryStore.currentState
    const context = monitorStore.buildRAGContext(currentState)
    analysisResult.value = ''  // 清空，准备流式写入

    await queryRAG(prompt, context, (chunk) => {
      analysisResult.value += chunk
    })
  } catch (err) {
    console.warn('RAG 查询失败，降级到本地分析:', err)
    analysisResult.value = runLocalAnalysis(type, prompt)
      + '\n\n<span class="tag-warn">⚠ AI 服务暂不可用，已使用本地规则分析</span>'
  } finally {
    isAnalyzing.value = false
  }
}

// 原有分析逻辑重命名为 fallback
function runLocalAnalysis(type, prompt) {
  switch (type) {
    case 'efficiency': return buildEfficiencyAnalysis()
    case 'bottleneck': return buildBottleneckAnalysis()
    case 'machine_load': return buildMachineLoadAnalysis()
    case 'transfer_analysis': return buildTransferAnalysis()
    case 'event_summary': return buildEventSummary()
    case 'custom': return buildCustomAnalysis(prompt)
    default: return '未知分析类型'
  }
}
```

### 4.5 RAG_BACKEND_URL 运行时注入

Docker 传入的是 `RAG_BACKEND_URL`，但 Vite 只注入 `VITE_` 前缀变量。

**方案（推荐）: 运行时动态获取**

在 frontend 的 Dockerfile entrypoint 或 nginx 配置中，启动时将环境变量写入 `index.html`：

```bash
# frontend Dockerfile entrypoint 片段
# 用 envsubst 替换 index.html 中的占位符
envsubst '${RAG_BACKEND_URL}' < /app/index.html.template > /app/index.html
```

`index.html` 中：
```html
<script>
  window.__RAG_BACKEND_URL__ = '${RAG_BACKEND_URL}';
</script>
```

`rag.js` 中读取：
```javascript
const RAG_URL = window.__RAG_BACKEND_URL__ || 'http://localhost:8000'
```

优点：修改 RAG 地址只需重启容器，无需重新构建镜像。

```python
from dataclasses import dataclass, field, asdict
from typing import Optional, List
import json, os

@dataclass
class RunRecord:
    id: str
    factory_type: str
    algorithm: str = ''
    config_name: str = ''
    start_time: str = ''
    end_time: str = ''
    duration_seconds: float = 0
    status: str = 'running'
    total_steps: int = 0
    final_efficiency: float = 0
    final_utilization: float = 0
    event_count: int = 0
    error_count: int = 0
    data_file: str = ''

@dataclass
class RunData:
    """单次运行完整数据（存储为 JSON）"""
    meta: dict
    metrics_timeline: list = field(default_factory=list)
    events: list = field(default_factory=list)
    key_snapshots: list = field(default_factory=list)

    def save(self, filepath: str):
        os.makedirs(os.path.dirname(filepath), exist_ok=True)
        with open(filepath, 'w', encoding='utf-8') as f:
            json.dump(asdict(self), f, ensure_ascii=False, indent=2)

    @classmethod
    def load(cls, filepath: str) -> 'RunData':
        with open(filepath, 'r', encoding='utf-8') as f:
            data = json.load(f)
        return cls(**data)
```

**`manager.py`** — 核心管理类：

```python
import sqlite3
import json
import os
from datetime import datetime
from pathlib import Path
from .models import RunRecord, RunData

class HistoryManager:
    def __init__(self, data_dir: str = "data/history"):
        self.data_dir = Path(data_dir)
        self.db_path = self.data_dir / "history.db"
        self.runs_dir = self.data_dir / "runs"
        self._init_db()

    def _init_db(self):
        self.runs_dir.mkdir(parents=True, exist_ok=True)
        with sqlite3.connect(str(self.db_path)) as conn:
            conn.executescript("""
                CREATE TABLE IF NOT EXISTS runs (...);  -- 见 5.1 表结构
                CREATE TABLE IF NOT EXISTS run_metrics_summary (...);
                CREATE INDEX IF NOT EXISTS idx_runs_start_time ON runs(start_time);
                CREATE INDEX IF NOT EXISTS idx_runs_factory_type ON runs(factory_type);
            """)

    def create_run(self, factory_type: str, algorithm: str = '',
                   config_name: str = '') -> RunRecord:
        """创建新运行记录"""
        run_id = f"run_{int(datetime.now().timestamp() * 1000)}"
        record = RunRecord(
            id=run_id, factory_type=factory_type,
            algorithm=algorithm, config_name=config_name,
            start_time=datetime.now().isoformat(),
            status='running', data_file=f"runs/{run_id}.json",
        )
        with sqlite3.connect(str(self.db_path)) as conn:
            conn.execute(
                "INSERT INTO runs (...) VALUES (?, ?, ?, ?, ?, ?, ?)",
                (record.id, record.factory_type, record.algorithm,
                 record.config_name, record.start_time, record.status,
                 record.data_file)
            )
        return record

    def complete_run(self, run_id: str, metrics_summary: dict = None,
                     run_data: RunData = None):
        """完成运行，保存数据"""
        now = datetime.now().isoformat()
        if run_data:
            run_data.meta['end_time'] = now
            filepath = self.runs_dir / f"{run_id}.json"
            run_data.save(str(filepath))
```

**`routes.py`** — API 路由：

```python
from fastapi import APIRouter, Query, Body
from .manager import HistoryManager

router = APIRouter(prefix="/history", tags=["history"])
manager = HistoryManager()

@router.get("/runs")
async def list_runs(limit: int = 20, offset: int = 0, factory_type: str = None):
    """获取运行记录列表"""
    runs = manager.list_runs(limit=limit, offset=offset, factory_type=factory_type)
    return {"status": "ok", "runs": runs, "count": len(runs)}

@router.get("/run/{run_id}")
async def get_run(run_id: str):
    """获取单次运行详情"""
    run = manager.get_run(run_id)
    if not run:
        return {"status": "error", "message": f"运行记录不存在: {run_id}"}
    return {"status": "ok", "data": run}

@router.get("/run/{run_id}/logs")
async def get_run_logs(run_id: str, event_type: str = None, limit: int = 100):
    """获取运行日志"""
    logs = manager.get_run_logs(run_id, event_type=event_type, limit=limit)
    return {"status": "ok", "logs": logs, "count": len(logs)}

@router.post("/compare")
async def compare_runs(body: dict = Body(...)):
    """对比多次运行"""
    run_ids = body.get("run_ids", [])
    if len(run_ids) < 2:
        return {"status": "error", "message": "至少需要 2 个运行 ID"}
    result = manager.compare_runs(run_ids)
    return {"status": "ok", "data": result}

@router.delete("/run/{run_id}")
async def delete_run(run_id: str):
    """删除运行记录"""
    manager.delete_run(run_id)
    return {"status": "ok", "message": f"运行 {run_id} 已删除"}

@router.get("/stats")
async def history_stats():
    """历史统计摘要"""
    runs = manager.list_runs(limit=1000)
    return {
        "status": "ok",
        "total_runs": len(runs),
        "factory_types": list(set(r['factory_type'] for r in runs)),
    }
```
