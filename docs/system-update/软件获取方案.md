# Docker 构建工具二进制获取方案

`application/dockerfile/docker-bin/` 中的三个二进制用于在容器内执行 docker compose 命令和 Python 包管理。
均为 Linux x86_64 静态编译，单文件即可运行。

## 当前版本

| 工具 | 版本 | 用途 |
|------|------|------|
| `docker` | 28.1.1 | Docker CLI，DockerProxy 通过它执行 `docker compose` |
| `docker-compose` | v2.35.1 | Docker Compose 插件 |
| `uv` | 0.11.3 | Python 包管理器，Dockerfile 中 `uv sync` 安装依赖 |

## 获取方式

### 1. Docker CLI

官方发布页：https://download.docker.com/linux/static/stable/x86_64/

```bash
# 指定版本下载（替换版本号即可）
DOCKER_VERSION="28.1.1"

# 方式 A：官方源
curl -fsSL "https://download.docker.com/linux/static/stable/x86_64/docker-${DOCKER_VERSION}.tgz" -o docker.tgz

# 方式 B：TUNA 镜像
curl -fsSL "https://mirrors.tuna.tsinghua.edu.cn/docker-ce/linux/static/stable/x86_64/docker-${DOCKER_VERSION}.tgz" -o docker.tgz

# 解压，只需 docker 二进制（包含 docker cli 和 docker proxy 等）
tar xzf docker.tgz docker/docker
mv docker/docker application/dockerfile/docker-bin/docker
chmod +x application/dockerfile/docker-bin/docker
rm -rf docker docker.tgz
```

验证：
```bash
application/dockerfile/docker-bin/docker --version
# Docker version 28.1.1, build 4eba377
```

### 2. Docker Compose 插件

官方发布页：https://github.com/docker/compose/releases

```bash
COMPOSE_VERSION="v2.35.1"

# 方式 A：GitHub 直连
curl -fsSL "https://github.com/docker/compose/releases/download/${COMPOSE_VERSION}/docker-compose-linux-x86_64" \
  -o application/dockerfile/docker-bin/docker-compose

# 方式 B：ghproxy 加速（国内网络）
curl -fsSL "https://ghfast.top/https://github.com/docker/compose/releases/download/${COMPOSE_VERSION}/docker-compose-linux-x86_64" \
  -o application/dockerfile/docker-bin/docker-compose

chmod +x application/dockerfile/docker-bin/docker-compose
```

验证：
```bash
application/dockerfile/docker-bin/docker-compose version
# Docker Compose version v2.35.1
```

### 3. uv

官方发布页：https://github.com/astral-sh/uv/releases

```bash
UV_VERSION="0.11.3"

# 方式 A：官方安装脚本（自动检测最新版）
curl -LsSf https://astral.sh/uv/install.sh | sh

# 方式 B：指定版本直接下载二进制
curl -fsSL "https://github.com/astral-sh/uv/releases/download/${UV_VERSION}/uv-x86_64-unknown-linux-gnu.tar.gz" -o uv.tar.gz
tar xzf uv.tar.gz "uv-x86_64-unknown-linux-gnu/uv"
mv "uv-x86_64-unknown-linux-gnu/uv" application/dockerfile/docker-bin/uv
chmod +x application/dockerfile/docker-bin/uv
rm -rf uv-x86_64-unknown-linux-gnu uv.tar.gz

# 方式 C：ghproxy 加速
curl -fsSL "https://ghfast.top/https://github.com/astral-sh/uv/releases/download/${UV_VERSION}/uv-x86_64-unknown-linux-gnu.tar.gz" -o uv.tar.gz
```

验证：
```bash
application/dockerfile/docker-bin/uv --version
# uv 0.11.3 (x86_64-unknown-linux-gnu)
```

## 版本升级

升级时只需替换版本号重新下载即可。选择版本的原则：

| 工具 | 版本选择建议 |
|------|------------|
| `docker` | 使用 stable 通道最新版，参考 https://docs.docker.com/release-notes/ |
| `docker-compose` | 与 docker CLI 版本兼容即可，一般用最新版 |
| `uv` | 用最新版，参考 https://docs.astral.sh/uv/getting-started/installation/ |

## 完整更新脚本

一键更新三个二进制：

```bash
#!/bin/bash
set -e

BIN_DIR="application/dockerfile/docker-bin"
DOCKER_VERSION="28.1.1"
COMPOSE_VERSION="v2.35.1"
UV_VERSION="0.11.3"

echo ">>> 更新 docker ${DOCKER_VERSION}"
curl -fsSL "https://mirrors.tuna.tsinghua.edu.cn/docker-ce/linux/static/stable/x86_64/docker-${DOCKER_VERSION}.tgz" | tar xz -C /tmp docker/docker
mv /tmp/docker/docker "${BIN_DIR}/docker"

echo ">>> 更新 docker-compose ${COMPOSE_VERSION}"
curl -fsSL "https://ghfast.top/https://github.com/docker/compose/releases/download/${COMPOSE_VERSION}/docker-compose-linux-x86_64" -o "${BIN_DIR}/docker-compose"

echo ">>> 更新 uv ${UV_VERSION}"
curl -fsSL "https://github.com/astral-sh/uv/releases/download/${UV_VERSION}/uv-x86_64-unknown-linux-gnu.tar.gz" | tar xz -C /tmp "uv-x86_64-unknown-linux-gnu/uv"
mv "/tmp/uv-x86_64-unknown-linux-gnu/uv" "${BIN_DIR}/uv"

chmod +x "${BIN_DIR}/docker" "${BIN_DIR}/docker-compose" "${BIN_DIR}/uv"
rm -rf /tmp/docker /tmp/uv-x86_64-unknown-linux-gnu

echo ">>> 验证"
"${BIN_DIR}/docker" --version
"${BIN_DIR}/docker-compose" version
"${BIN_DIR}/uv" --version
echo "完成"
```

## 从零配置（新环境部署）

在项目根目录下执行，直接下载到目标位置：

```bash
#!/bin/bash
set -e

BIN_DIR="application/dockerfile/docker-bin"
mkdir -p "${BIN_DIR}"
cd "${BIN_DIR}"

# 版本号（按需修改）
DOCKER_VERSION="28.1.1"
COMPOSE_VERSION="v2.35.1"
UV_VERSION="0.11.3"

# --- Docker CLI ---
echo ">>> 下载 docker ${DOCKER_VERSION}"
curl -fsSL "https://mirrors.tuna.tsinghua.edu.cn/docker-ce/linux/static/stable/x86_64/docker-${DOCKER_VERSION}.tgz" \
  | tar xz -C /tmp docker/docker
cp /tmp/docker/docker .
rm -rf /tmp/docker

# --- Docker Compose ---
echo ">>> 下载 docker-compose ${COMPOSE_VERSION}"
curl -fsSL "https://ghfast.top/https://github.com/docker/compose/releases/download/${COMPOSE_VERSION}/docker-compose-linux-x86_64" \
  -o docker-compose

# --- uv ---
echo ">>> 下载 uv ${UV_VERSION}"
curl -fsSL "https://github.com/astral-sh/uv/releases/download/${UV_VERSION}/uv-x86_64-unknown-linux-gnu.tar.gz" \
  | tar xz -C /tmp "uv-x86_64-unknown-linux-gnu/uv"
cp /tmp/uv-x86_64-unknown-linux-gnu/uv .
rm -rf /tmp/uv-x86_64-unknown-linux-gnu

# --- 权限 ---
chmod +x docker docker-compose uv

# --- 验证 ---
echo ""
echo "========== 验证 =========="
./docker --version
./docker-compose version
./uv --version
echo "==========================="
echo "配置完成，文件位于: ${BIN_DIR}/"
```

预期输出：
```
Docker version 28.1.1, build 4eba377
Docker Compose version v2.35.1
uv 0.11.3 (x86_64-unknown-linux-gnu)
配置完成，文件位于: application/dockerfile/docker-bin/
```

**文件结构：**
```
application/dockerfile/docker-bin/
├── docker            # Docker CLI (43MB)
├── docker-compose    # Compose 插件 (73MB)
└── uv                # Python 包管理器 (59MB)
```

## 注意事项

- 这三个二进制仅用于 **Dockerfile 构建阶段** COPY 进容器，宿主机上不需要替换系统已有的版本
- `docker-compose` 在容器内作为 Docker CLI 插件使用，路径固定为 `/usr/local/lib/docker/cli-plugins/docker-compose`
- 如果架构不是 x86_64（如 ARM 服务器），需要下载对应架构的版本（替换 URL 中的 `x86_64` 为 `aarch64`）
- 国内网络下 docker 用 TUNA 镜像，compose/uv 用 ghproxy 加速；海外环境可去掉加速前缀直连 GitHub
