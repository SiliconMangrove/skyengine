# 可配置指标看板（DashboardPanel）设计

> 范围：前端"指标 tab"如何做到**不写代码就能加图表**。文档面向想新增指标卡片的同事。
>
> 关联代码：
> - `application/frontend/src/components/DashboardPanel.vue`（渲染器）
> - `application/frontend/src/components/charts/`（图表模板：line / bar）
> - `application/frontend/src/hooks/metrics/`（数据 hook 注册中心）
> - `application/frontend/src/config/dashboards/*.json`（配置文件）
>
> 日期：2026-07-05

---

## 0. 一句话总结

**加一张图 = 改一个 JSON + （如果是新指标）写一个 hook 文件**。

整条链路：

```
dashboard JSON (配置)
   ↓ 列出 hook + template
DashboardPanel.vue (渲染器)
   ↓ 按 hook id 查表
hooks/metrics/<id>.js (纯函数：ctx → series)
   ↓ 从 monitorStore / factoryStore 拿数据
返回 [{name, data:[{x,y}]}]
   ↓
charts/<template>.vue (echarts 模板：line/bar)
   ↓ 渲染
```

**关键设计**：hook 是纯函数，对数据来源无感知；图表模板对指标名无感知。两者通过 JSON 配置组合。

---

## 1. 三个核心概念

### 1.1 Hook（数据钩子）
位置：`src/hooks/metrics/<id>.js`

职责：从 `ctx` 派生出一组 `{x, y}` 数据点。**不画图、不知道图形**。

契约：
```js
export default {
  id: 'machine_util',        // 必须全局唯一，与 dashboard JSON 的 hook 字段对应
  label: '机器利用率',        // fallback 标题
  unit: '%',                  // 可选
  multiSeries: false,         // 可选，true 表示返回多 series（折线对比/分组柱）
  series: (ctx) => [{x, y}, ...]   // 或多 series 模式返回 [{name, data, color?}, ...]
}
```

`ctx` 由 DashboardPanel 归一化注入：
```js
{
  metrics: monitorStore.metricsTimeline,   // [{step, metrics:{...}, metrics_reward}, ...]
  frames:  factoryStore.historyBuffer,     // [snapshot, ...]
  events:  monitorStore.events,            // [event, ...]
  source:  'live'                          // 'live' | 'archive'
}
```

### 1.2 Template（图表模板）
位置：`src/components/charts/<name>.vue`

职责：把 `series` 渲染成 echarts 图。**不知道指标含义**。

注册：`src/components/charts/index.js`
```js
import LineChart from './LineChart.vue'
import BarChart  from './BarChart.vue'
export const TEMPLATES = { line: LineChart, bar: BarChart }
```

支持的 props：
- `series: [{name, data:[{x,y}], color?}]`
- `y-label`、`y-max`、`mode`（bar 用 `grouped`）、`height`

### 1.3 配置文件 JSON
位置：`src/config/dashboards/<factory>.json`

示例（`static_factory.json`）：
```json
{
  "id": "static_factory_default",
  "title": "StaticFactory 实时看板",
  "layout": "grid-2col",
  "cards": [
    { "hook": "machine_util",   "template": "line", "title": "机器利用率",  "yMax": 100, "unit": "%" },
    { "hook": "machine_load",   "template": "bar",  "title": "各机器负载",  "mode": "grouped" },
    { "hook": "agv_stats",      "template": "line", "title": "AGV 综合指标", "multiSeries": true }
  ]
}
```

字段：
| 字段 | 必需 | 说明 |
|---|---|---|
| `id` | ✓ | 配置唯一标识（导出/存档用） |
| `title` |  | 看板大标题 |
| `layout` |  | `grid-2col`（默认，自适应填充） |
| `cards[]` | ✓ | 卡片列表，每个卡片 = 1 hook + 1 template |
| `cards[].hook` | ✓ | hook id（查 `HOOKS` 表） |
| `cards[].template` | ✓ | `line` / `bar` |
| `cards[].title` |  | 卡片标题（不填用 hook.label） |
| `cards[].unit` |  | Y 轴单位 |
| `cards[].yMax` |  | Y 轴上限 |
| `cards[].mode` |  | bar 模式：`grouped` |
| `cards[].multiSeries` |  | 是否多 series 对比 |

## 2. 工作机制（运行时）

```
DashboardPanel.vue mounted
    │
    ├─ resolveTemplate(card.template)   → charts/index.js 查表
    ├─ getHook(card.hook)               → hooks/metrics/index.js 查表
    └─ ctx = computed(() => ({
         metrics: monitorStore.metricsTimeline,  ← 响应式
         frames:  factoryStore.historyBuffer,
         events:  monitorStore.events,
         source:  'live',
       }))
       │
       ↓ (monitorStore push 一条新 metrics → ctx 重算)
    safeSeries(card) = hook.series(ctx)
       │
       ↓
    <component :is="resolvedTemplate" :series="safeSeries" />
       │
       ↓ echarts 重绘
```

**响应式链路**：每次 SSE 推一条 metrics / state frame，Pinia store 更新 → `ctx` computed 触发 → hook 重算 → 图表自动刷新。

数据来源切换：
- **实时**（DashboardPanel）：`source: 'live'`，store = monitorStore + factoryStore
- **存档回放**（AnalysisDetailView）：`source: 'archive'`，store = run 的历史快照

同一套 hook 在两种模式都能用 —— 这是 hook 抽象的关键收益。

## 3. 新增一个指标卡片（操作步骤）

### 场景 A：用已有 hook，只是想换张图（最简单）
直接改 JSON，不写代码：
```json
{ "hook": "machine_util", "template": "bar", "title": "利用率柱状图" }
```

### 场景 B：指标字段已有，但 hook 没写
1. 新建 `src/hooks/metrics/<id>.js`：
   ```js
   export default {
     id: 'swap_conflict',
     label: '路径冲突数',
     series: (ctx) => (ctx.metrics || []).map(p => ({
       x: p.step,
       y: p.metrics?.swap_conflict_count ?? 0,
     })),
   }
   ```
2. 在 `src/hooks/metrics/index.js` 注册：
   ```js
   import swapConflict from './swap_conflict.js'
   export const HOOKS = { ..., swap_conflict: swapConflict }
   ```
3. JSON 里加卡片：
   ```json
   { "hook": "swap_conflict", "template": "line", "title": "路径冲突" }
   ```

### 场景 C：需要新图表模板（如饼图）
1. 新建 `src/components/charts/PieChart.vue`（参考 BarChart 实现，接收 `series` props）
2. 在 `src/components/charts/index.js` 注册：`pie: PieChart`
3. JSON 用 `"template": "pie"`

## 4. 当前已实现的 hook 一览

| hook id | label | 数据源 | 多 series | 说明 |
|---|---|---|---|---|
| `machine_util` | 机器利用率 | metrics | 否 | `machine_utilization` × 100% |
| `efficiency` | 系统效率 | metrics | 否 | 系统级效率指标 |
| `agv_activity` | AGV 活动数 | frames | 否 | 当前 active AGV 计数 |
| `agv_stats` | AGV 综合指标 | metrics | 是 | loaded/busy/waiting 三条线 |
| `job_progress` | Job 完成进度 | frames | 否 | 已完成 / 总数 |
| `machine_load` | 各机器负载 | frames | 是 | 每机器一柱，load 或 queue_length 降级 |
| `event_timeline` | 事件累计 | events | 是 | 按 etype 分类的累计折线 |

**降级策略**（machine_load 为例）：
- StaticFactory frame 有 `machines[].load`（0-100%）→ 直接用
- 容器化 frame 无 load 字段，只有 `queue_length` → 自动降级用 queue_length
- 都没有 → 空曲线（不报错）

## 5. 关键约束 & 调试

### 5.1 数据形状约束
hook 必须返回 **`[{x, y}, ...]`**（单 series）或 **`[{name, data:[{x,y}], color?}, ...]`**（多 series）。乱返回会被 `safeSeries()` 吞掉并 `console.warn`。

### 5.2 console 调试
- `[DashboardPanel] unknown hook: <id>` → JSON 里 hook 名打错 / 忘了在 `HOOKS` 注册
- `[DashboardPanel] unknown template: <name>` → template 名打错 / 忘了在 `TEMPLATES` 注册
- `[DashboardPanel] hook <id> error: ...` → hook 函数抛异常（多半是字段路径错）

### 5.3 性能
- 每次 SSE 推送都会触发所有 hook 重算。当前规模（≤ 500 step × 7 卡片）无感。
- 如果指标点数上千还卡，考虑 hook 内部 memoize 或 cards 内部抽样。

## 6. 与现有 MetricsPanel 的关系

`FactoryTabsPanel.vue` 默认在 metrics tab 渲染内置 `MetricsPanel`（旧组件，写死的指标列表）。
工厂视图可以通过 `#tab-metrics` slot **覆盖**默认渲染：

```vue
<FactoryTabsPanel ...>
  <template #tab-metrics>
    <DashboardPanel :config="dashboardConfig" />
  </template>
</FactoryTabsPanel>
```

StaticFactory 已迁移到 DashboardPanel。DockerFactory 还在用默认 MetricsPanel —— 下一步把 DashboardPanel 也接到 DockerFactory 即可（hook 都是通用的，无需改）。

## 7. 给后端 metrics 字段开发者的提示

要让一个新指标能被看板消费，只需在 metrics point 的 `metrics` 子字典里多吐一个字段：
```jsonc
{
  "step": 42,
  "metrics": {
    "machine_utilization": 0.42,
    "your_new_field": 123        // ← 加这行
  },
  "metrics_reward": 0.0
}
```
然后写一个对应 hook（场景 B），就能画图了。**不需要改前端 store、SSE 通路、DashboardPanel**。
