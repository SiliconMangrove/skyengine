# 大禹系统：日志分析 + 绘图自定义 功能完整拆解
结合聊天记录里师兄的讲解，分**绘图自定义（在线可视化）**、**日志分析模块（华佗离线系统）**两部分讲清设计思路、实现逻辑、核心要求。

## 一、绘图自定义（在线可视化模块）
### 1. 核心设计思想：数据与显示模板完全分离
这是自定义绘图最关键的规范，不能把数据、图表写死耦合在一起。
- **数据层**：原始业务指标、日志指标（要展示的数值、时间序列、统计量）
- **显示模板层**：前端固化好的图表渲染模板（曲线图、柱状图、饼图、折线图等各类Chart）
两者解耦后，才能实现自由搭配、灵活自定义。

### 2. 三层实现架构与开发要求
#### （1）数据提取层：Hook 扩展函数
要求：
1. 所有指标数据提取逻辑统一封装为**Hook钩子函数**；
2. 开放扩展能力，新增业务指标不用改底层绘图代码，只新增对应Hook即可；
3. Hook只负责一件事：从系统数据源/实时流里捞出需要展示的结构化数据，不包含任何图表渲染逻辑。

#### （2）前端模板层：预定义图表组件
要求：
1. 前端提前写好全套可视化模板（各类Chart基础渲染逻辑固化在前端）；
2. 模板只定义外观、坐标轴、图例、渲染样式，**不绑定固定数据源**；
3. 模板可复用：同一套折线模板，既能展示CPU时序，也能展示请求耗时，不用重复开发绘图代码。

#### （3）配置组合层：配置文件自由编排（自定义核心）
这就是“绘图自定义”的用户侧能力，核心要求：
1. 提供统一配置文件（JSON/YAML等）作为中间桥梁；
2. 配置内支持**任意组合**：指定「哪个Hook输出的数据」+「使用哪套前端图表模板」；
3. 业务/运维人员修改配置，就能实时切换图表类型、切换展示指标，无需前后端发版改代码；
4. 支持多图表拼接、多指标叠加在一张图、分面板展示等自定义布局。

### 3. 对比“直接写死Chart”的优势（解决提问人的疑惑）
如果直接把数据硬编码写进Chart：新增指标、换图表样式、调整展示维度都要改前端代码；
数据模板分离架构：只改配置+新增Hook，快速实现自定义可视化，扩展性极强。

## 二、日志分析模块（配套华佗离线系统）
### 1. 定位：离线日志分析绘图，和在线绘图共用一套可视化逻辑
在线大禹系统看实时数据，华佗是**离线事后分析**，两者绘图底层逻辑统一，降低重复开发成本。

### 2. 完整流程与功能要求
1. **日志导出输入**
   支持把线上全量运行日志导出为离线文件，作为模块输入源；
2. **日志解析提取**
   内置日志解析规则，也可复用上面的Hook体系：从原始日志文本里过滤、提取结构化指标（耗时、报错量、QPS、延迟等）；
3. **复用统一绘图内核**
   提取出结构化数据后，使用和大禹在线完全一致的图表渲染逻辑、前端模板；
4. **离线交互能力**
   支持离线图表拖拽、时间区间自由选取（聊天里提到“只拽任意时刻”），离线也能缩放、筛选、对比任意时间段日志指标；
5. 区分边界：
   - 大禹：在线实时指标、在线自定义看板；
   - 华佗日志分析：离线日志文件导入、事后复盘分析绘图，底层可视化逻辑打通复用。

## 三、两个功能点统一的硬性设计要求汇总
### 通用约束
1. 可视化逻辑复用：在线绘图、离线日志绘图共用同一套Chart模板与渲染规则；
2. 数据/视图解耦：严禁数据逻辑和图表渲染代码强耦合；
3. 可扩展：指标新增走Hook扩展，图表新增仅开发前端模板，不改动核心流程。

### 绘图自定义专项要求
1. 支持配置化编排指标与图表，零代码调整可视化看板；
2. 数据源抽象为Hook函数，统一数据输出格式；
3. 前端图表模板无硬编码数据源，纯样式渲染单元。

### 日志分析专项要求
1. 离线日志文件导入、解析、结构化提取能力；
2. 复用大禹绘图引擎输出离线图表；
3. 离线图表支持时间拖拽、区间筛选交互；
4. 适配大批量历史日志的离线批量分析场景。

## 四、通俗举例帮你理解
举个业务场景：想做一张“每秒请求量折线图”
1. Hook函数：抓取日志/监控里的QPS时序数据（只输出数字序列）；
2. 前端模板：通用折线Chart（只定义线条、坐标轴怎么画）；
3. 配置文件：绑定「QPS Hook + 折线模板」，页面自动渲染；
- 在线：大禹实时拉取Hook数据，在线看；
- 离线：华佗导入昨天的日志，运行同一个Hook提取历史QPS，用同一折线模板画图，拖拽看凌晨/高峰时段数据。

如果后续想改成柱状图对比每小时QPS，只需要修改配置文件，把模板换成柱状图即可，不用改提取数据的Hook、不用重写绘图代码，这就是绘图自定义的价值。
