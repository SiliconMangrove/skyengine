# 地图覆盖层实现计划：路径轨迹 / 热力图 / 区域标注

## Context

在工厂可视化地图上增加三个可切换覆盖层，让仿真指标直接在地图上呈现。当前渲染使用 Three.js (FactoryVisualization3D.vue)，工具栏在 FactoryPlayerSSE.vue 的 `#header` slot 中浮动显示。数据来源于 `store.historyBuffer`（帧历史）和 `store.currentState`（当前状态）。

## 核心设计决策

1. **状态位置**：覆盖层开关放在 `FactoryPlayerSSE.vue` 的局部 ref 中（非全局 store），通过 props 传入 3D 组件
2. **渲染方式**：
   - 轨迹：Three.js `Line` + 预分配 `BufferGeometry`，从 historyBuffer 聚合 AGV 历史位置，带尾迹渐变（越靠近当前位置越不透明，越远越透明）
   - 热力图：每个网格单元一个半透明 `PlaneGeometry`，颜色按 AGV 访问频率映射（蓝→绿→黄→红），仅统计 AGV 经过频率
   - 区域标注：复用已有的 `labels-overlay` HTML 层，投影区域质心坐标到屏幕空间，显示区域名 + AGV 数 + 设备利用率
3. **性能**：轨迹预分配最大 5000 点；热力图在切换开启时计算一次，currentIndex 变化时增量更新；区域标注随渲染循环自然刷新

- 传递三个新 props 给 `<FactoryVisualization3D>`

### 3. `application/frontend/src/components/FactoryVisualization3D.vue`（主要工作量）

**新增 props**：`showTrajectory`, `showHeatmap`, `showRegionAnnotation`（Boolean, default false）

**新增 Three.js 组**（在 `initScene()` 中）：
- `trajectoryGroup` — 轨迹线组
- `heatmapGroup` — 热力图网格组

#### 轨迹叠加（带尾迹渐变）

- `initTrajectory(agvCount)` — 为每个 AGV 创建预分配 `THREE.Line`，使用 `ShaderMaterial` 实现透明度渐变：
  - 顶点 attribute 添加 `alpha` 分量（0.0~1.0）
  - Vertex shader 传递 alpha 到 fragment
  - Fragment shader 中用 `gl_FragColor.a = vAlpha` 控制透明度
  - 越靠近当前位置（最近帧）alpha 越大（1.0），越远越小（趋近 0.1）
  - 视觉效果：AGV 当前位置附近轨迹清晰，远端轨迹淡出消失
- `updateTrajectory()` — 从 `historyBuffer[0..currentIndex]` 读取各 AGV 位置写入线几何，同时更新 alpha attribute
- `disposeTrajectory()` — 清理几何和材质

#### 热力图叠加

- `initHeatmap()` — 为每个网格单元创建半透明平面（共享 PlaneGeometry 实例，独立材质）
- `computeHeatmapData(endIdx)` — 扫描 historyBuffer[0..endIdx] 统计每格 AGV 访问频次
- `updateHeatmapColors()` — 对数归一化后映射蓝→绿→黄→红色阶
- `disposeHeatmap()` — 清理所有网格和材质

#### 区域标注

- 在 `updateScreenLabels()` 中扩展：当 `showRegionAnnotation` 为 true 时，对每个非装饰区域计算：
  - 区域内 AGV 数量（匹配 positions_xy 与 area 边界框）
  - 区域内机器利用率（WORKING 数 / 总数）
- 用 `.region-label` CSS 类区分样式（半透明背景 + 金色边框）

#### 渲染循环集成（在 `updateFromStore()` 中）

- 根据 props 控制各 group 的 visible 属性
- 轨迹：visible 变为 true 时初始化 + 首次更新；currentIndex 变化时调用 updateTrajectory
- 热力图：visible 变为 true 时初始化 + 计算；currentIndex 变化时重新计算
- 标注：已由 updateScreenLabels 自然驱动

#### watch 监听

- `showTrajectory`：off → disposeTrajectory()；on → initTrajectory() + updateTrajectory()
- `showHeatmap`：off → disposeHeatmap()；on → initHeatmap() + computeHeatmapData() + updateHeatmapColors()
- `showRegionAnnotation`：触发 updateScreenLabels()

#### 清理

- `onBeforeUnmount` 中调用 disposeTrajectory / disposeHeatmap
- `buildStaticElements` 重建时重置叠加层状态

## 实施顺序

1. `assets.js` — 添加颜色常量（零风险）
2. `FactoryPlayerSSE.vue` — 添加开关和 props 传递（零风险，props 默认 false）
3. `FactoryVisualization3D.vue` — 按轨迹 → 热力图 → 区域标注顺序实现
4. 可选：添加 `.region-label` 样式

## 验证方式

1. 启动前端 `npm run dev`，打开任意工厂页面
2. 加载配置并启动仿真，积累帧数据
3. 点击工具栏三个覆盖层按钮，验证：
   - 轨迹：AGV 移动路径以彩色线条显示，带尾迹渐变效果
   - 热力图：地图网格以热力颜色填充，频繁经过区域偏红
   - 区域标注：各区域上方浮现标签，显示 AGV 数和利用率
4. 可同时开启多个覆盖层
5. 关闭覆盖层时地图恢复正常
