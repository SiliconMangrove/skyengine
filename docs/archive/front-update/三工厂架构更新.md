# 三工厂管理视图重构方案

## 背景

当前三个工厂管理视图存在大量重复代码：

| 视图 | 文件 | 行数 | 布局 | 执行模式 |
|------|------|------|------|----------|
| DockerFactoryManage | `views/factory/DockerFactoryManage.vue` | ~320 行 | 浮动工具栏 + RightSidePanel | backendSystemTest (3步 API) |
| GridFactoryManage | `views/factory/GridFactoryManage.vue` | ~400 行 | DraggablePanel + Tabs | backendSystemTest |
| StaticFactoryManage | `views/factory/StaticFactoryManage.vue` | ~340 行 | DraggablePanel + Tabs | runFullSystemTest (前端) |

**Grid 与 Static 的模板和脚本 ~90% 相同**，包括：
- 完全相同的 DraggablePanel + 5 Tab 结构
- 完全相同的算法选择 UI（FJSP / MAPF / Assigner 下拉框）
- ~100 行完全相同的 CSS（`.dp-tabs`、`.simulation-tab`、`.plan-select` 等）
- 相同的 Tab 切换逻辑、面板开关逻辑

**三个视图共享的逻辑**：
- 算法选项管理（fjspOptions / mapfOptions / assignerOptions）
- 算法字符串拼接（`fjsp+mapf+assigner`）
- 执行状态管理（`isRunningTest`、`stopTest`）
- Store 初始化与清理
- `onUnmounted` 清理模式

---

## 重构策略：Composable + 共享组件

### 层级 1: `composables/useSimulationConfig.js`

提取三个视图共享的**算法配置与执行逻辑**：

```js
/**
 * @param {Object} options
 * @param {Object} options.defaults - 默认选中值 { fjsp, mapf, assigner }
 * @param {Object} options.defaultOptions - 默认选项列表 { fjsp: [], mapf: [], assigner: [] }
 * @param {boolean} options.fetchFromApi - 是否从 /algo 接口获取选项
 * @param {'backend'|'frontend'|'docker'} options.executionMode - 执行模式
 */
export function useSimulationConfig(options)
```

**返回值**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `fjspOptions` | `Ref<Array>` | FJSP 算法选项 |
| `mapfOptions` | `Ref<Array>` | MAPF 路由选项 |
| `assignerOptions` | `Ref<Array>` | 任务分配选项 |
| `selectedFjsp` | `Ref<string>` | 当前选中 FJSP |
| `selectedMapf` | `Ref<string>` | 当前选中 MAPF |
| `selectedAssigner` | `Ref<string>` | 当前选中 Assigner |
| `isRunningTest` | `Ref<boolean>` | 是否正在执行 |
| `algorithmString` | `ComputedRef<string>` | 拼接后的算法字符串 |
| `handleExecutePlan(store, monitorStore, opts)` | `Function` | 统一执行入口 |
| `handleStop()` | `Function` | 停止当前仿真 |
| `fetchAlgoOptions()` | `Function` | 从 API 获取算法选项 |
| `cleanup()` | `Function` | 清理（用于 onUnmounted） |

**执行模式差异**：

| 模式 | 执行流程 | 使用方 |
|------|----------|--------|
| `frontend` | 调用 `runFullSystemTest()`，前端本地模拟 | StaticFactoryManage |
| `backend` | 调用 `backendSystemTest({ algorithm })` | GridFactoryManage |
| `docker` | 先 API 设定策略 → 重置 → 启动容器 → `backendSystemTest` | DockerFactoryManage |

### 层级 2: `components/FactoryTabsPanel.vue`

提取 Grid 与 Static 共享的 **DraggablePanel + Tabs 布局**：

**Props**：

| Prop | 类型 | 说明 |
|------|------|------|
| `tabs` | `Array<{key, label, icon}>` | **由父组件定义**的 tab 列表，方便扩展 |
| `modelValue` | `string` | 当前激活 tab（v-model 绑定） |
| `isEditMode` | `boolean` | 编辑模式状态 |
| `isRunningTest` | `boolean` | 执行中状态 |
| `showPanel` | `boolean` | 面板显示/隐藏 |

**Events**：

| Event | Payload | 说明 |
|-------|---------|------|
| `update:modelValue` | `string` | tab 切换 |
| `update:showPanel` | `boolean` | 面板关闭 |
| `edit-mode-change` | `boolean` | ConfigPanel 编辑模式变化 |
| `execute` | - | 执行按钮点击 |
| `stop` | - | 停止按钮点击 |

**Slots（按 tab key 动态命名）**：

| Slot | 默认内容 | 说明 |
|------|----------|------|
| `tab-simulation` | 无默认 | 仿真 tab 全部内容，父组件自定义 |
| `tab-control` | `<ControlPanel>` | 控制 tab |
| `tab-config` | `<ConfigPanel>` | 配置 tab |
| `tab-metrics` | `<MetricsPanel>` | 指标 tab |
| `tab-events` | `<EventPanel>` | 日志 tab |

**使用示例**（后续添加 agent 分析 tab）：

```vue
<FactoryTabsPanel
  :tabs="[
    { key: 'simulation', label: '仿真', icon: '🚀' },
    { key: 'control', label: '控制', icon: '⚙️' },
    { key: 'config', label: '配置', icon: '🔧' },
    { key: 'metrics', label: '指标', icon: '📊' },
    { key: 'events', label: '日志', icon: '📋' },
    { key: 'agent', label: 'AI分析', icon: '🤖' },  <!-- 新增 -->
  ]"
  v-model="activeTab"
>
  <template #tab-simulation>
    <!-- 各工厂自定义仿真内容 -->
  </template>
  <template #tab-agent>
    <!-- 未来的智能体分析面板 -->
  </template>
</FactoryTabsPanel>
```

### 层级 3: 共享 CSS 整合

将以下重复 CSS 从 Grid/Static 视图移入 `views/styles/FactoryManage.css`：

- `.dp-tabs` / `.dp-tab-btn` — Tab 栏样式（~40 行）
- `.simulation-tab` / `.sim-field` — 仿真 Tab 布局（~20 行）
- `.plan-select` — 算法下拉框（~25 行）
- `.launch-btn` / `.stop-btn` — 执行/停止按钮（~30 行）
- `.sim-btn-row` — 按钮行布局（~5 行）

## 重构后各视图结构

### StaticFactoryManage.vue（~60 行）

```vue
<template>
  <div class="factory-manage-container">
    <div class="middle-panel">
      <FactoryPlayerSSE :hide-control-panel="true" :edit-mode="isEditMode" />
    </div>
    <FactoryTabsPanel
      :tabs="tabs"
      v-model="activeTab"
      v-model:show-panel="showPanel"
      :is-edit-mode="isEditMode"
      :is-running-test="sim.isRunningTest.value"
      @edit-mode-change="isEditMode = $event"
      @execute="handleExecute"
      @stop="sim.handleStop()"
    >
      <template #tab-simulation>
        <SimulationControls
          :options="sim" :is-running="sim.isRunningTest.value"
          :show-stop="true"
          @execute="handleExecute" @stop="sim.handleStop()"
        />
      </template>
    </FactoryTabsPanel>
  </div>
</template>
```

### GridFactoryManage.vue（~80 行）

```vue
<template>
  <div class="factory-manage-container">
    <div class="middle-panel">
      <FactoryPlayerSSE :hide-control-panel="true" :edit-mode="isEditMode"
        :background-theme="backgroundTheme" :background-size="backgroundSize" />
    </div>
    <FactoryTabsPanel :tabs="tabs" v-model="activeTab" ...>
      <template #tab-simulation>
        <SimulationControls ... />
        <!-- Grid 独有的背景设置 -->
        <div class="sim-field">
          <label>场景风格</label>
          <select v-model="backgroundTheme" class="plan-select">...</select>
        </div>
      </template>
    </FactoryTabsPanel>
  </div>
</template>
```

### DockerFactoryManage.vue（~150 行）

```vue
<template>
  <div class="factory-manage-container">
    <div class="left-panel"><ControlPanel :disabled="isEditMode" /></div>
    <div class="middle-panel">
      <FactoryPlayerSSE ... />
      <div class="floating-toolbar">
        <!-- 使用 sim composable 的选项 -->
        <select v-model="sim.selectedFjsp.value">...</select>
        <select v-model="sim.selectedMapf.value">...</select>
        <select v-model="sim.selectedAssigner.value">...</select>
```

## 可扩展性：未来 Agent 分析 Tab

添加 AI 分析面板只需两步：

1. 在对应工厂视图的 `tabs` 数组中添加一项：
```js
{ key: 'agent', label: 'AI分析', icon: '🤖' }
```

2. 在 `FactoryTabsPanel` 中提供 `#tab-agent` slot 内容：
```vue
<template #tab-agent>
  <AgentAnalysisPanel />  <!-- 新组件 -->
</template>
```

无需修改 `FactoryTabsPanel` 或 `useSimulationConfig` 的代码。
