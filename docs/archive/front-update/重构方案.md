# GridFactory 前端重构方案

## 一、现状分析

### 当前布局

```
┌──────────┬────────────────────────────┬──────────────┐
│          │                            │              │
│ Control  │   Factory Visualization    │   Right      │
│ Panel    │   (FactoryPlayerSSE)       │   Side       │
│ (280px)  │                            │   Panel      │
│ 固定宽度  │   + Floating Toolbar       │   (Config,   │
│          │   + API Test Buttons       │   Metrics,   │
│          │                            │   Events)    │
│          │                            │   固定宽度    │
└──────────┴────────────────────────────┴──────────────┘
```

### 问题

1. **工厂画面被两侧面板挤压**，ControlPanel 280px + RightSidePanel 各占空间，中间可视化区域受限
2. **API 测试按钮组**（1️⃣2️⃣3️⃣）是调试用的，不应占据正式 UI 空间
3. **三旋钮算法选择**藏在 toolbar right 中，与正式控制面板功能重叠
4. **GridFactoryManage 和 DockerFactoryManage 高度相似**（~80% 代码重复），维护成本高

## 二、重构策略

> 核心思路：先做 DraggablePanel 通用组件，再逐步将所有面板迁移上去。

### 分阶段

| 阶段 | 内容 | 状态 |
|------|------|------|
| **Phase 1** | 创建 DraggablePanel 组件 | ✅ 已完成 |
| **Phase 2** | 用 DraggablePanel 重构 GridFactoryManage 布局 | ✅ 已完成 |
| **Phase 3** | 清理旧样式，统一透明高科技风格 | 待实施 |

## 三、Phase 1: DraggablePanel 组件（已完成）

### 文件

| 文件 | 说明 |
|------|------|
| `components/DraggablePanel.vue` | 可拖拽浮动面板组件 |
| `components/styles/DraggablePanel.css` | 玻璃态高科技样式 |

### API

```vue
<DraggablePanel
  title="控制面板"         <!-- 标题栏文字 -->
  icon="🎮"               <!-- 可选图标 -->
  :width="280"             <!-- 面板宽度 -->
  :initial-pos="{ x: 12, y: 60 }"  <!-- 初始位置 -->
  :default-collapsed="false"        <!-- 是否默认折叠 -->
  :max-height="600"                  <!-- 最大高度 (0=不限) -->
  @close="showPanel = false"         <!-- 关闭事件 -->
  @move="onMove"                     <!-- 拖拽结束事件 -->
>
  <!-- 任意面板内容 -->
  <ControlPanel />

  <!-- 折叠态摘要 (可选) -->
  <template #hint>12 台设备运行中</template>
</DraggablePanel>
```

### 功能

- 标题栏拖拽移动（支持鼠标和触摸）
- 折叠/展开切换
- 关闭按钮
- 边界约束（不会拖出屏幕）
- 玻璃态透明风格：`backdrop-filter: blur(16px)` + rgba 半透明背景
- hover 微光边框、拖拽中发光效果
- 进场动画

## 四、Phase 2: 重构 GridFactoryManage（待实施）

### 目标布局

```
┌──────────────────────────────────────────────────────────────┐
│                                                               │
│  ┌──────────────────┐                                        │
│  │ Draggable Panel   │        工厂可视化 (100%)               │
│  │ ┌────────────────┐│                                       │
│  │ │ [仿真] [控制] [配置] [指标] [日志]  ← Tab 栏            ││
│  │ ├────────────────┤│                                       │
│  │ │                ││                                       │
│  │ │  [仿真] Tab:   ││  FJSP▼  MAPF▼  Assigner▼            │
│  │ │  算法三旋钮 +   ││  [🚀 启动仿真]                      │
│  │ │  启动按钮       ││                                       │
│  │ │                ││                                       │
│  │ │  [控制] Tab:   ││  播放/暂停/重置 + 时间轴              │
│  │ │  ControlPanel  ││  实时状态 (AGV/机器/任务)             │
│  │ │                ││                                       │
│  │ │  [配置] Tab:   ││  ConfigPanel 内容                    │
│  │ │                ││                                       │
│  │ │  [指标] Tab:   ││  MetricsPanel 图表                   │
│  │ │                ││                                       │
│  │ │  [日志] Tab:   ││  EventPanel 日志列表                 │
│  │ └────────────────┘│                                       │
│  └──────────────────┘↑                                       │
│   可拖拽可折叠，单面板                                        │
│   Tab: 仿真 | 控制 | 配置 | 指标 | 日志                       │
│                                                               │
└──────────────────────────────────────────────────────────────┘
```

核心理念：**只保留一个 DraggablePanel**，工具栏（算法选择 + 启动按钮）作为"仿真"Tab 也收入面板内，最大化工厂可视化面积，界面上没有独立悬浮工具栏。

### 改动清单

#### 1. `FactoryManage.css` — 布局改为全屏

```css
/* 之前 */
.factory-manage-container { display: flex; height: 100vh; }
.left-panel { width: 280px; }

/* 之后 */
.factory-manage-container {
  position: relative;
  width: 100vw;
  height: 100vh;
  overflow: hidden;
}
.middle-panel { position: absolute; inset: 0; }
/* .left-panel 不再使用，删除 */
```

#### 2. `GridFactoryManage.vue` — 单个 DraggablePanel + Tab 切换

```vue
<template>
  <div class="factory-manage-container">
    <!-- 工厂可视化 (全屏) -->
    <div class="middle-panel">
      <FactoryPlayerSSE :hide-control-panel="true" :edit-mode="isEditMode" />
    </div>

    <!-- 唯一浮动面板: Tab 切换 (含工具栏) -->
    <DraggablePanel
      v-if="showPanel"
      :title="currentTabLabel"
      :icon="currentTabIcon"
      :width="300"
      :initial-pos="{ x: 12, y: 12 }"
      @close="showPanel = false"
    >
      <div class="dp-tabs">
        <button v-for="tab in tabs" :key="tab.key"
          :class="{ active: activeTab === tab.key }"
          @click="activeTab = tab.key">
          {{ tab.icon }} {{ tab.label }}
        </button>
      </div>

      <!-- 仿真 Tab: 算法选择 + 启动 -->
      <div v-if="activeTab === 'simulation'" class="simulation-tab">
        <select v-model="selectedFjsp">...</select>
        <select v-model="selectedMapf">...</select>
        <select v-model="selectedAssigner">...</select>
        <button @click="handleExecutePlan">🚀 启动仿真</button>
      </div>

      <ControlPanel v-if="activeTab === 'control'" :disabled="isEditMode" />
      <ConfigPanel v-if="activeTab === 'config'" ... />
      <MetricsPanel v-if="activeTab === 'metrics'" :show-chart="true" />
      <EventPanel v-if="activeTab === 'events'" />
    </DraggablePanel>

    <!-- 边缘切换按钮 -->
    <button v-if="!showPanel" class="panel-toggle left" @click="showPanel = true">
      ▶ 面板
    </button>
  </div>
</template>
```

#### 3. 移除 API 测试按钮组

删除 `api-test-toolbar` 整段模板和对应样式。`handleExecutePlan` 已完整覆盖。

#### 4. 移除独立浮动工具栏

原来的 `floating-toolbar-wrapper` 整段删除，算法三旋钮和启动按钮作为"仿真"Tab 移入 DraggablePanel 内部。

## 五、Phase 3: 样式统一（待实施）

所有 Tab 内的子面板需要适配暗色透明风格：

| 面板 | 改动 |
|------|------|
| ControlPanel.css | 白色背景 → 透明，文字色 → 浅色，滑块/按钮适配暗色 |
| ConfigPanel.css | 同上 |
| MetricsPanel.css | 图表背景透明 |
| EventPanel.css | 日志背景透明 |
| Tab 栏样式 | 新增 `.dp-tabs` 暗色 Tab 按钮样式 |

原则：DraggablePanel 提供玻璃容器，子面板只关心自身内容样式，背景/边框交给 DraggablePanel。
