# 0604 — 容器化系统 API 整理

## 1. 系统架构概览

```
前端 (Vue.js)
  │
  ▼ SSE + REST
SkyEngine 后端 (FastAPI)
  │
  ▼ DockerProxy (容器管理 + HTTP 代理)
  │
  ├─ engine 容器 (sim_server.py :8080)   ← 仿真运行时
  ├─ mapf 容器  (算法服务 :8001)         ← 路径规划求解
  └─ fjsp 容器  (算法服务 :8002)         ← 作业调度求解
```

DockerProxy 不直接跑仿真，它通过 HTTP 调用 sim_server 的 API 来控制仿真生命周期，并透传 SSE 流给前端。

## 2. sim_server.py — 当前 API（容器内 :8080）

### 2.1 仿真控制

| 方法 | 路径 | 请求体 | 响应 | 说明 |
|------|------|--------|------|------|
| POST | `/sim/create` | 见下方 sim_config | `{"status": "ok"}` | 创建仿真环境 |
| POST | `/sim/play` | 无 | `{"status": "ok"}` | 启动/继续仿真 |
| POST | `/sim/pause` | 无 | `{"status": "ok"}` | 暂停仿真 |
| POST | `/sim/reset` | 无 | `{"status": "ok"}` | 重置环境 |
| POST | `/sim/stop` | 无 | `{"status": "ok"}` | 停止仿真 |
| GET | `/sim/state` | 无 | `{"step": N, "running": bool}` | 查询当前状态 |

系统启动之后，DockerProxy 会调用 `/sim/create` 传入配置，随后调用 `/sim/play` 启动仿真。前端可以通过 `/sim/state` 查询当前仿真步数和运行状态。
其中算法镜像依然需要sim_server进行启动，启动后再进行仿真。

### 2.4 `/sim/create` 请求体 (sim_config)

```json
{
  "problem_instance": "问题结构体", // 这个结构体包含 FJSP/MAPF 实例数据，需要你直接读这个数据，包括machine位置、map大小等信息。
  "mapf_algorithm": "astar",             // 路径规划算法: astar / mapf_gpt / ... ，需要启动对应的算法镜像
  "fjsp_algorithm": "pso",              // 作业调度算法: pso / de / drl / best 等，需要启动对应的算法镜像
  "solver_assign": "nearest",             // 分配策略: nearest / random / ...
}
```

## 3. DockerProxy — 当前 API（宿主机端）

### 3.1 对外暴露的生命周期方法

| 方法 | 阶段 | 说明 |
|------|------|------|
| `set_config(config: dict)` | 配置 | 保存前端传来的工厂配置 |
| `set_algorithm(algorithm: str)` | 配置 | 设置算法组合，如 `"pso+astar+nearest"` |
| `get_algorithm() -> str` | 查询 | 获取当前算法 |
| `get_initialized() -> bool` | 查询 | 是否已初始化 |
| `initialize()` | Phase 1 | 创建 Docker 网络 + 启动 engine 容器 + 等待就绪 |
| `start()` | Phase 2 | 启动 mapf/fjsp 容器 → POST `/sim/create` → POST `/sim/play` |
| `pause()` | 运行中 | POST engine `/sim/pause` |
| `reset()` | 运行中 | POST engine `/sim/reset` |
| `stop()` | 运行中 | POST engine `/sim/stop` + 清理所有容器 |
| `cleanup()` | 清理 | 停止并移除所有容器 + 网络 |

### 3.2 对外暴露的 SSE/数据接口

| 方法 | 说明 |
|------|------|
| `get_state_events() -> list` | 返回当前状态快照（非流式，本地拼装） |
| `get_metrics_events() -> list` | 返回当前指标快照 |
| `get_control_events() -> list` | 返回控制状态 |
| `get_state_snapshot() -> dict` | 状态快照 |
| `get_metrics_snapshot() -> dict` | 指标快照 |
| `get_control_status() -> dict` | 控制状态 |
| `state_stream()` (async gen) | 透传 sim_server `/stream/state` SSE |
| `metrics_stream()` (async gen) | 透传 sim_server `/stream/metrics` SSE |

### 3.3 状态判断

| 方法 | 说明 |
|------|------|
| `is_running() -> bool` | 仿真是否在运行 |
| `is_paused() -> bool` | 是否暂停 |
| `is_idle() -> bool` | 是否空闲 |

### 3.4 `initialize()` 做的事

1. `_cleanup_all()` — 清理残留容器/网络
2. 创建 Docker bridge 网络 `{project}_sim-net`
3. 启动 engine 容器（挂载 dataset + sim_server.py + sky_executor）
4. `_discover_engine()` — 轮询获取容器 IP（最多 30s）
5. `_wait_for_health()` — 等待 `/health` 返回 200（最多 60s）

### 3.5 `start()` 做的事

1. 从前端 config 中读取 `fjsp_algorithm` 和 `mapf_algorithm`，选择对应镜像
2. 启动 mapf 容器 + fjsp 容器（接入同一网络）
3. 构造 sim_config（含 `problem_instance` 实际数据），POST 到 engine `/sim/create`
4. POST 到 engine `/sim/play`
5. 标记 `_streaming = True`

### 3.6 传入 sim_server 的 sim_config（应改为）

```json
{
  "problem_instance": {
    // FJSP 数据：jobs、machines 等
    "fjsp_data": { ... },
    // MAPF 数据：grid map、machine 位置、AGV 起终点等
    "mapf_data": { ... },
    // 仿真参数
    "num_agv": 4,
    "seed": 42,
    "max_steps": 1000,
    "obs_radius": 5,
    "obs_type": "default"
  },
  "mapf_algorithm": "astar",        // 选择 MAPF 算法 → 决定启动哪个 mapf 容器镜像
  "fjsp_algorithm": "pso",          // 选择 FJSP 算法 → 决定启动哪个 fjsp 容器镜像
  "solver_assign": "nearest"        // 分配策略: nearest / random
}
```

> 注意：`mapf_algorithm` 和 `fjsp_algorithm` 由 DockerProxy 读取后决定启动哪个算法容器，
> sim_server 收到后用于配置 Coordinator 的求解方式。
> 不再传文件名（`fjsp_instance`/`mapf_instance`）和服务地址（`mapf_service_url`/`fjsp_service_url`）。
> DockerProxy 负责容器间通信，sim_server 不需要知道算法容器的地址。

### 3.7 DockerProxy 内部状态

| 属性 | 类型 | 说明 |
|------|------|------|
| `status` | ExecutionStatus | idle / running / paused / stopped / error |
| `current_step` | int | 当前仿真步数（从 SSE 帧解析） |
| `initialized` | bool | Phase 1 是否完成 |
| `_streaming` | bool | 是否在 SSE 透传中 |
| `_algorithm` | str | 算法组合字符串 |
| `_algorithm_parts` | dict | 拆解后的 {fjsp, mapf, assigner} |
| `_config` | dict | 前端传入的工厂配置 |
| `_engine_url` | str | engine 容器内网 IP URL |
| `_engine_container` | Container | engine 容器引用 |
| `_mapf_container` | Container | mapf 容器引用 |
| `_fjsp_container` | Container | fjsp 容器引用 |
| `_network` | Network | Docker bridge 网络 |

### 3.8 容器/镜像映射

| 角色 | 容器名 | 镜像 | 端口 |
|------|--------|------|------|
| engine | `skyengine-online-engine` | `skyengine:latest` | 8080 |
| mapf | `skyengine-online-mapf` | `skyengine-mapf-{algo}:latest` | 8001 |
| fjsp | `skyengine-online-fjsp` | `skyengine-fjsp-{algo}:latest` | 8002 |

算法镜像选项:
- fjsp: pso / de / drl / best
- mapf: astar / mapf_gpt
